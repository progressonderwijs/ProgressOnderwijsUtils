<p><b>HTML5</b><sup id="cite_ref-name_4-0" class="reference"><a href="#cite_note-name-4">[a]</a></sup> is a <a href="/wiki/Markup_language" title="Markup language">markup language</a> used for structuring and presenting content on the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>. It is the fifth and current version of the <a href="/wiki/HTML" title="HTML">HTML</a> standard.</p>
<p>It was published in October 2014 by the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C)<sup id="cite_ref-finalars_2-1" class="reference"><a href="#cite_note-finalars-2">[2]</a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[4]</a></sup> to improve the language with support for the latest multimedia, while keeping it both easily readable by humans and consistently understood by computers and devices such as <a href="/wiki/Web_browser" title="Web browser">web browsers</a>, <a href="/wiki/Parsing" title="Parsing">parsers</a>, etc. HTML5 is intended to subsume not only <a href="/wiki/HTML_4" title="HTML 4" class="mw-redirect">HTML 4</a>, but also <a href="/wiki/XHTML" title="XHTML">XHTML</a> 1 and <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM Level 2 HTML</a>.<sup id="cite_ref-HTML5-20110405_6-0" class="reference"><a href="#cite_note-HTML5-20110405-6">[5]</a></sup></p>
<p>HTML5 includes detailed processing models to encourage more interoperable implementations; it extends, improves and rationalizes the markup available for documents, and introduces markup and <a href="/wiki/Application_programming_interfaces" title="Application programming interfaces" class="mw-redirect">application programming interfaces</a> (APIs) for complex <a href="/wiki/Web_application" title="Web application">web applications</a>.<sup id="cite_ref-HTML5diffHTML4_7-0" class="reference"><a href="#cite_note-HTML5diffHTML4-7">[6]</a></sup> For the same reasons, HTML5 is also <a href="/wiki/HTML5_in_mobile_devices" title="HTML5 in mobile devices">a candidate for cross-platform mobile applications</a>, because it includes features designed with low-powered devices in mind.</p>
<p>Many new <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntactic</a> features are included. To natively include and handle <a href="/wiki/Multimedia" title="Multimedia">multimedia</a> and <a href="/wiki/2D_computer_graphics" title="2D computer graphics">graphical</a> content, the new <code><a href="/wiki/HTML5_video" title="HTML5 video">&lt;video&gt;</a></code>, <code><a href="/wiki/HTML5_Audio" title="HTML5 Audio">&lt;audio&gt;</a></code> and <code><a href="/wiki/Canvas_element" title="Canvas element">&lt;canvas&gt;</a></code> <a href="/wiki/HTML_element" title="HTML element">elements</a> were added, and support for scalable vector graphics (<a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a>) content and <a href="/wiki/MathML" title="MathML">MathML</a> for mathematical formulas. To enrich the <a href="/wiki/Semantic_web" title="Semantic web" class="mw-redirect">semantic</a> content of documents, new page structure elements such as <code>&lt;main&gt;</code>, <code>&lt;section&gt;</code>, <a href="/wiki/Article_element_(HTML5)" title="Article element (HTML5)"><code>&lt;article&gt;</code></a>, <code>&lt;header&gt;</code>, <code>&lt;footer&gt;</code>, <code>&lt;aside&gt;</code>, <code>&lt;nav&gt;</code> and <code>&lt;figure&gt;</code>, are added. New <a href="/wiki/HTML_attribute" title="HTML attribute">attributes</a> are introduced, some elements and attributes have been removed, and others such as <code>&lt;a&gt;</code>, <code>&lt;cite&gt;</code> and <code>&lt;menu&gt;</code> have been changed, redefined or standardized.</p>
<p>The APIs and <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) are now fundamental parts of the HTML5 specification<sup id="cite_ref-HTML5diffHTML4_7-1" class="reference"><a href="#cite_note-HTML5diffHTML4-7">[6]</a></sup> and HTML5 also better defines the processing for any invalid documents.<sup id="cite_ref-:0_8-0" class="reference"><a href="#cite_note-:0-8">[7]</a></sup></p>

<h2><span id="History" class="mw-headline">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The <a href="/wiki/Web_Hypertext_Application_Technology_Working_Group" title="Web Hypertext Application Technology Working Group" class="mw-redirect">Web Hypertext Application Technology Working Group</a> (WHATWG) began work on the new standard in 2004. At that time, HTML 4.01 had not been updated since 2000,<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[8]</a></sup> and the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C) was focusing future developments on <a href="/wiki/XHTML_2.0" title="XHTML 2.0" class="mw-redirect">XHTML 2.0</a>. In 2009, the W3C allowed the XHTML 2.0 Working Group's charter to expire and decided not to renew it.<sup id="cite_ref-:1_10-0" class="reference"><a href="#cite_note-:1-10">[9]</a></sup> W3C and WHATWG are currently working together on the development of HTML5.<sup id="cite_ref-:1_10-1" class="reference"><a href="#cite_note-:1-10">[9]</a></sup></p>
<p>The <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla Foundation</a> and <a href="/wiki/Opera_Software" title="Opera Software">Opera Software</a> presented a position paper at a World Wide Web Consortium (W3C) workshop in June 2004,<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[10]</a></sup> focusing on developing technologies that are backward compatible with existing browsers,<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[11]</a></sup> including an initial draft specification of Web Forms 2.0. The workshop concluded with a vote—8 for, 14 against—for continuing work on HTML.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[12]</a></sup> Immediately after the workshop, the Web Hypertext Application Technology Working Group (WHATWG) was formed to start work based upon that position paper, and a second draft, Web Applications 1.0, was also announced.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[13]</a></sup> The two specifications were later merged to form HTML5.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[14]</a></sup> The HTML5 specification was adopted as the starting point of the work of the new HTML working group of the W3C in 2007.</p>
<p><a href="/wiki/WHATWG" title="WHATWG">WHATWG</a> published the First Public Working Draft of the specification on 22 January 2008.<sup id="cite_ref-HTML5_16-0" class="reference"><a href="#cite_note-HTML5-16">[15]</a></sup></p>
<h3><span id=".22Thoughts_on_Flash.22" class="mw-headline">"Thoughts on Flash"</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=2" title="Edit section: &quot;Thoughts on Flash&quot;">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>While some features of HTML5 are often compared to <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a>, the two technologies are very different. Both include features for playing audio and video within web pages, and for using <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">Scalable Vector Graphics</a>. However, HTML5 on its own cannot be used for animation or interactivity – it must be supplemented with <a href="/wiki/CSS3" title="CSS3" class="mw-redirect">CSS3</a> or <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>. There are many Flash capabilities that have no direct counterpart in HTML5. See <a href="/wiki/Comparison_of_HTML5_and_Flash" title="Comparison of HTML5 and Flash">Comparison of HTML5 and Flash</a>. HTML5’s interactive capabilities became a topic of mainstream media around April 2010<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">[16]</a></sup><sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[17]</a></sup><sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[18]</a></sup><sup id="cite_ref-20" class="reference"><a href="#cite_note-20">[19]</a></sup> after <a href="/wiki/Apple_Inc" title="Apple Inc" class="mw-redirect">Apple Inc</a>'s then-CEO <a href="/wiki/Steve_Jobs" title="Steve Jobs">Steve Jobs</a> issued a public letter titled "<a href="/wiki/Apple_and_Adobe_Flash_controversy" title="Apple and Adobe Flash controversy">Thoughts on Flash</a>" where he concluded that "Flash is no longer necessary to watch video or consume any kind of web content" and that "new open standards created in the mobile era, such as HTML5, will win".<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[20]</a></sup> This sparked a debate in web development circles suggesting that, while HTML5 provides enhanced functionality, developers must consider the varying browser support of the different parts of the standard as well as other functionality differences between HTML5 and Flash.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[21]</a></sup> In early November 2011, Adobe announced that it would discontinue development of Flash for mobile devices and reorient its efforts in developing tools using HTML5.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[22]</a></sup></p>
<h3><span id="Last_Call.2C_Candidate_and_Recommendation" class="mw-headline">Last Call, Candidate and Recommendation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=3" title="Edit section: Last Call, Candidate and Recommendation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On 14 February 2011, the W3C extended the charter of its HTML Working Group with clear milestones for HTML5. In May 2011, the working group advanced HTML5 to "Last Call", an invitation to communities inside and outside W3C to confirm the technical soundness of the specification. The W3C developed a comprehensive test suite to achieve broad interoperability for the full specification by 2014, which was the target date for recommendation.<sup id="cite_ref-w3c2014_24-0" class="reference"><a href="#cite_note-w3c2014-24">[23]</a></sup> In January 2011, the WHATWG renamed its "HTML5" living standard to "HTML". The W3C nevertheless continued its project to release HTML5.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">[24]</a></sup></p>
<p>In July 2012, WHATWG and W3C decided on a degree of separation. W3C will continue the HTML5 specification work, focusing on a single definitive standard, which is considered as a "snapshot" by WHATWG. The WHATWG organization will continue its work with HTML5 as a "Living Standard". The concept of a living standard is that it is never complete and is always being updated and improved. New features can be added but functionality will not be removed.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[25]</a></sup></p>
<p>In December 2012, W3C designated HTML5 as a Candidate Recommendation.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[26]</a></sup> The criterion for advancement to <a href="/wiki/W3C_recommendation#W3C_Recommendation_.28REC.29" title="W3C recommendation" class="mw-redirect">W3C Recommendation</a> is "two 100% complete and fully interoperable implementations".<sup id="cite_ref-W3Crec_28-0" class="reference"><a href="#cite_note-W3Crec-28">[27]</a></sup><sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[28]</a></sup></p>
<p>On 16 September 2014, W3C moved HTML5 to Proposed Recommendation.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[29]</a></sup></p>
<p>On 28 October 2014, HTML5 was released as a stable W3C Recommendation,<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[30]</a></sup> bringing the specification process to completion.<sup id="cite_ref-finalars_2-2" class="reference"><a href="#cite_note-finalars-2">[2]</a></sup></p>
<p>On 1 November 2016, HTML5.1 was released as a stable W3C Recommendation.<sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[31]</a></sup></p>

<h2><span id="Features_and_APIs" class="mw-headline">Features and APIs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=5" title="Edit section: Features and APIs">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The W3C proposed a greater reliance on modularity as a key part of the plan to make faster progress, meaning identifying specific features, either proposed or already existing in the spec, and advancing them as separate specifications. Some technologies that were originally defined in HTML5 itself are now defined in separate specifications:</p>
<ul>
<li>HTML Working Group – HTML Canvas 2D Context;</li>
<li>Web Apps Working Group – <a href="/wiki/Web_Messaging" title="Web Messaging">Web Messaging</a>, <a href="/wiki/Web_Workers" title="Web Workers" class="mw-redirect">Web Workers</a>, <a href="/wiki/Web_Storage" title="Web Storage" class="mw-redirect">Web Storage</a>, <a href="/wiki/WebSocket" title="WebSocket">WebSocket</a>, <a href="/wiki/Server-sent_events" title="Server-sent events">Server-sent events</a>, <a href="/wiki/Web_Components" title="Web Components">Web Components</a><sup id="cite_ref-34" class="reference"><a href="#cite_note-34">[33]</a></sup> (this was not part of HTML5 though); Note that the Web Applications Working Group was closed in October 2015 and its deliverables transferred to the Web Platform Working Group (WPWG).</li>
<li>IETF HyBi Working Group – WebSocket Protocol;</li>
<li>WebRTC Working Group – <a href="/wiki/WebRTC" title="WebRTC">WebRTC</a>;</li>
<li>Web Media Text Tracks Community Group – <a href="/wiki/WebVTT" title="WebVTT">WebVTT</a>.</li>
</ul>
<p>After the standardization of the HTML5 specification in October 2014,<sup id="cite_ref-35" class="reference"><a href="#cite_note-35">[34]</a></sup> the core vocabulary and features are being extended in four ways.<sup id="cite_ref-36" class="reference"><a href="#cite_note-36">[35]</a></sup> Likewise, some features that were removed from the original HTML5 specification have been standardized separately as modules, such as <a href="/wiki/Microdata_(HTML)" title="Microdata (HTML)">Microdata</a> and <a href="/wiki/Canvas_element" title="Canvas element">Canvas</a>. Technical specifications introduced as HTML5 extensions such as <a href="/wiki/Polyglot_markup" title="Polyglot markup">Polyglot Markup</a> have also been standardized as modules. Some W3C specifications that were originally separate specifications have been adapted as HTML5 extensions or features, such as SVG. Some features that might have slowed down the standardization of HTML5 will be standardized as upcoming specifications, instead. HTML 5.1 is expected to be finalized in 2016, and it is currently on the standardization track at the W3C.</p>
<h2><span id="Features" class="mw-headline">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=6" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span id="Markup" class="mw-headline">Markup</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=7" title="Edit section: Markup">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>HTML5 introduces <a href="/wiki/HTML_element" title="HTML element">elements</a> and attributes that reflect typical usage on modern <a href="/wiki/Website" title="Website">websites</a>. Some of them are semantic replacements for common uses of generic block (<code class="nowrap">&lt;div&gt;</code>) and inline (<code class="nowrap">&lt;span&gt;</code>) elements, for example <code class="nowrap">&lt;nav&gt;</code> (website navigation block), <code class="nowrap">&lt;footer&gt;</code> (usually referring to bottom of web page or to last lines of HTML code), or <code class="nowrap">&lt;audio&gt;</code> and <code class="nowrap">&lt;video&gt;</code> instead of <code class="nowrap">&lt;object&gt;</code>.<sup id="cite_ref-37" class="reference"><a href="#cite_note-37">[36]</a></sup><sup id="cite_ref-38" class="reference"><a href="#cite_note-38">[37]</a></sup><sup id="cite_ref-39" class="reference"><a href="#cite_note-39">[38]</a></sup> Some deprecated elements from <a href="/wiki/HTML_4.01" title="HTML 4.01" class="mw-redirect">HTML 4.01</a> have been dropped, including purely presentational elements such as <code class="nowrap">&lt;font&gt;</code> and <code class="nowrap">&lt;center&gt;</code>, whose effects have long been superseded by the more capable <a href="/wiki/Cascading_Style_Sheet" title="Cascading Style Sheet" class="mw-redirect">Cascading Style Sheets</a>.<sup id="cite_ref-40" class="reference"><a href="#cite_note-40">[39]</a></sup> There is also a renewed emphasis on the importance of <a href="/wiki/DOM_scripting" title="DOM scripting" class="mw-redirect">DOM scripting</a> (e.g., JavaScript) in Web behavior.</p>
<p>The HTML5 syntax is no longer based on <a href="/wiki/Standard_Generalized_Markup_Language" title="Standard Generalized Markup Language">SGML</a><sup id="cite_ref-41" class="reference"><a href="#cite_note-41">[40]</a></sup><sup id="cite_ref-42" class="reference"><a href="#cite_note-42">[41]</a></sup> despite the similarity of its markup. It has, however, been designed to be backward compatible with common parsing of older versions of HTML. It comes with a new introductory line that looks like an SGML <a href="/wiki/Document_type_declaration" title="Document type declaration">document type declaration</a>, <code style="white-space:nowrap;">&lt;!DOCTYPE html&gt;</code>, which triggers the standards-compliant <a href="/wiki/Quirks_mode" title="Quirks mode">rendering mode</a>.<sup id="cite_ref-43" class="reference"><a href="#cite_note-43">[42]</a></sup> Since 5 January 2009, HTML5 also includes <i>Web Forms 2.0</i>, a previously separate <a href="/wiki/WHATWG" title="WHATWG">WHATWG</a> specification.<sup id="cite_ref-44" class="reference"><a href="#cite_note-44">[43]</a></sup><sup id="cite_ref-45" class="reference"><a href="#cite_note-45">[44]</a></sup></p>
<h3><span id="New_APIs" class="mw-headline">New APIs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=8" title="Edit section: New APIs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div style="width:422px;" class="thumbinner"><a href="/wiki/File:HTML5_APIs_and_related_technologies_taxonomy_and_status.svg" class="image"><img alt src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7f/HTML5_APIs_and_related_technologies_taxonomy_and_status.svg/420px-HTML5_APIs_and_related_technologies_taxonomy_and_status.svg.png" width="420" height="291" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/7f/HTML5_APIs_and_related_technologies_taxonomy_and_status.svg/630px-HTML5_APIs_and_related_technologies_taxonomy_and_status.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/7f/HTML5_APIs_and_related_technologies_taxonomy_and_status.svg/840px-HTML5_APIs_and_related_technologies_taxonomy_and_status.svg.png 2x" data-file-width="1133" data-file-height="786" class="thumbimage"></a>
<div class="thumbcaption">
<div class="magnify"><a href="/wiki/File:HTML5_APIs_and_related_technologies_taxonomy_and_status.svg" title="Enlarge" class="internal"></a></div>
HTML5 related APIs<sup id="cite_ref-46" class="reference"><a href="#cite_note-46">[45]</a></sup></div>
</div>
</div>
<p>In addition to specifying markup, HTML5 specifies scripting <a href="/wiki/Application_programming_interfaces" title="Application programming interfaces" class="mw-redirect">application programming interfaces</a> (APIs) that can be used with <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>.<sup id="cite_ref-47" class="reference"><a href="#cite_note-47">[46]</a></sup> Existing <a href="/wiki/Document_object_model" title="Document object model" class="mw-redirect">document object model</a> (DOM) interfaces are extended and <i><a href="/wiki/De_facto" title="De facto">de facto</a></i> features documented. There are also new APIs, such as:</p>
<ul>
<li><a href="/wiki/Canvas_element" title="Canvas element">Canvas</a>;<sup id="cite_ref-48" class="reference"><a href="#cite_note-48">[47]</a></sup></li>
<li>Timed Media Playback;<sup id="cite_ref-49" class="reference"><a href="#cite_note-49">[48]</a></sup></li>
<li>Offline;<sup id="cite_ref-50" class="reference"><a href="#cite_note-50">[49]</a></sup></li>
<li>Editable content;<sup id="cite_ref-51" class="reference"><a href="#cite_note-51">[50]</a></sup></li>
<li><a href="/wiki/Drag-and-drop" title="Drag-and-drop" class="mw-redirect">Drag-and-drop</a>;<sup id="cite_ref-52" class="reference"><a href="#cite_note-52">[51]</a></sup></li>
<li>History;<sup id="cite_ref-53" class="reference"><a href="#cite_note-53">[52]</a></sup></li>
<li><a href="/wiki/MIME_type" title="MIME type" class="mw-redirect">MIME type</a> and protocol handler registration;<sup id="cite_ref-54" class="reference"><a href="#cite_note-54">[53]</a></sup><sup id="cite_ref-55" class="reference"><a href="#cite_note-55">[54]</a></sup></li>
<li><a href="/wiki/Microdata_(HTML)" title="Microdata (HTML)">Microdata</a>;<sup id="cite_ref-56" class="reference"><a href="#cite_note-56">[55]</a></sup></li>
<li><a href="/wiki/Web_Messaging" title="Web Messaging">Web Messaging</a>;<sup id="cite_ref-57" class="reference"><a href="#cite_note-57">[56]</a></sup></li>
<li><a href="/wiki/Web_Storage" title="Web Storage" class="mw-redirect">Web Storage</a> – a key-value pair storage framework that provides behaviour similar to <a href="/wiki/HTTP_Cookie" title="HTTP Cookie" class="mw-redirect">cookies</a> but with larger storage capacity and improved API.<sup id="cite_ref-58" class="reference"><a href="#cite_note-58">[57]</a></sup></li>
</ul>
<p>Not all of the above technologies are included in the W3C HTML5 specification, though they are in the WHATWG HTML specification.<sup id="cite_ref-59" class="reference"><a href="#cite_note-59">[58]</a></sup> Some related technologies, which are not part of either the W3C HTML5 or the WHATWG HTML specification, are as follows. The W3C publishes specifications for these separately:</p>
<ul>
<li><a href="/wiki/W3C_Geolocation_API" title="W3C Geolocation API">Geolocation</a>;</li>
<li><a href="/wiki/Web_SQL_Database" title="Web SQL Database">Web SQL Database</a> – a local SQL Database (no longer maintained);<sup id="cite_ref-60" class="reference"><a href="#cite_note-60">[59]</a></sup></li>
<li><a href="/wiki/IndexedDB" title="IndexedDB" class="mw-redirect">IndexedDB</a> – an indexed hierarchical key-value store (formerly WebSimpleDB);<sup id="cite_ref-61" class="reference"><a href="#cite_note-61">[60]</a></sup></li>
<li><a href="/wiki/HTML5_File_API" title="HTML5 File API">File</a><sup id="cite_ref-62" class="reference"><a href="#cite_note-62">[61]</a></sup> – an API intended to handle file uploads and file manipulation;<sup id="cite_ref-63" class="reference"><a href="#cite_note-63">[62]</a></sup></li>
<li>Directories and System – an API intended to satisfy client-side-storage use cases not well served by databases;<sup id="cite_ref-64" class="reference"><a href="#cite_note-64">[63]</a></sup></li>
<li>File Writer – an API for writing to files from web applications;<sup id="cite_ref-65" class="reference"><a href="#cite_note-65">[64]</a></sup></li>
<li><a href="/wiki/HTML5_Audio#Web_Audio_API_and_MediaStream_Processing_API" title="HTML5 Audio">Web Audio</a><sup id="cite_ref-66" class="reference"><a href="#cite_note-66">[65]</a></sup> – a high-level JavaScript API for processing and synthesizing audio in web applications;</li>
<li>ClassList.<sup id="cite_ref-67" class="reference"><a href="#cite_note-67">[66]</a></sup></li>
<li><a href="/w/index.php?title=Web_Cryptography&amp;action=edit&amp;redlink=1" title="Web Cryptography (page does not exist)" class="new">Web Cryptography</a><sup id="cite_ref-68" class="reference"><a href="#cite_note-68">[67]</a></sup></li>
<li><a href="/wiki/WebRTC" title="WebRTC">WebRTC</a><sup id="cite_ref-69" class="reference"><a href="#cite_note-69">[68]</a></sup></li>
</ul>
<p>HTML5 cannot provide animation within web pages. Additional JavaScript or <a href="/wiki/CSS3" title="CSS3" class="mw-redirect">CSS3</a> functionality is necessary for animating HTML elements. Animation is also possible using JavaScript and HTML 4<sup id="cite_ref-70" class="reference"><a href="#cite_note-70">[69]</a></sup><sup style="white-space:nowrap;" class="noprint Inline-Template">[<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag failed verification of its source citation(s). (January 2011)">not in citation given</span></a></i>]</sup>, and within SVG elements through <a href="/wiki/Synchronized_Multimedia_Integration_Language" title="Synchronized Multimedia Integration Language">SMIL</a>, although browser support of the latter remains uneven as of 2011.</p>
<h3><span id="XHTML5_.28XML-serialized_HTML5.29" class="mw-headline">XHTML5 (XML-serialized HTML5)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=9" title="Edit section: XHTML5 (XML-serialized HTML5)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>XML documents must be served with an XML <a href="/wiki/Internet_media_type" title="Internet media type" class="mw-redirect">Internet media type</a> (often called "MIME type") such as <code>application/xhtml+xml</code> or <code>application/xml</code>,<sup id="cite_ref-71" class="reference"><a href="#cite_note-71">[70]</a></sup> and must conform to strict, well-formed syntax of XML. XHTML5 is simply XML-serialized HTML5 data (e.g. not having any unclosed tags), sent with one of XML media types. HTML that has been written to conform to both the HTML and XHTML specifications – and which will therefore produce the same DOM tree whether parsed as HTML or XML – is called <a href="/wiki/Polyglot_markup" title="Polyglot markup">polyglot markup</a>.<sup id="cite_ref-w3c_72-0" class="reference"><a href="#cite_note-w3c-72">[71]</a></sup></p>
<h3><span id="Error_handling" class="mw-headline">Error handling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=10" title="Edit section: Error handling">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>HTML5 is designed so that old browsers <a href="/wiki/Forward_compatibility" title="Forward compatibility">can safely ignore</a> new HTML5 constructs.<sup id="cite_ref-HTML5diffHTML4_7-2" class="reference"><a href="#cite_note-HTML5diffHTML4-7">[6]</a></sup> In contrast to HTML 4.01, the HTML5 specification gives detailed rules for <a href="/wiki/Lexing" title="Lexing" class="mw-redirect">lexing</a> and <a href="/wiki/Parsing" title="Parsing">parsing</a>, with the intent that compliant browsers will produce the same results when parsing incorrect syntax.<sup id="cite_ref-whatfaq_73-0" class="reference"><a href="#cite_note-whatfaq-73">[72]</a></sup> Although HTML5 now defines a consistent behavior for "<a href="/wiki/Tag_soup" title="Tag soup">tag soup</a>" documents, those documents are not regarded as conforming to the HTML5 standard.<sup id="cite_ref-whatfaq_73-1" class="reference"><a href="#cite_note-whatfaq-73">[72]</a></sup></p>
<h3><span id="Popularity" class="mw-headline">Popularity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=11" title="Edit section: Popularity">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>According to a report released on 30 September 2011, 34 of the world's top 100 Web sites were using HTML5 – the adoption led by <a href="/wiki/Search_engine" title="Search engine" class="mw-redirect">search engines</a> and <a href="/wiki/Social_network" title="Social network">social networks</a>.<sup id="cite_ref-74" class="reference"><a href="#cite_note-74">[73]</a></sup> Another report released in August 2013 has shown that 153 of the <i><a href="/wiki/Fortune_500" title="Fortune 500">Fortune 500</a></i> U.S. companies implemented HTML5 on their corporate websites.<sup id="cite_ref-75" class="reference"><a href="#cite_note-75">[74]</a></sup></p>
<p>Since 2014, HTML5 is at least <a href="/wiki/Comparison_of_layout_engines_(HTML5)" title="Comparison of layout engines (HTML5)">partially supported by most popular layout engines</a>.</p>
<h3><span id="Differences_from_HTML.C2.A04.01_and_XHTML.C2.A01.x" class="mw-headline">Differences from HTML 4.01 and XHTML 1.x</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=12" title="Edit section: Differences from HTML 4.01 and XHTML 1.x">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The following is a cursory list of differences and some specific examples.</p>
<ul>
<li>New parsing rules: oriented towards flexible parsing and compatibility; not based on SGML</li>
<li>Ability to use inline SVG and MathML in <code>text/html</code></li>
<li>New <a href="/wiki/HTML_element" title="HTML element">elements</a>: <code>article</code>, <code>aside</code>, <code>audio</code>, <code>bdi</code>, <code>canvas</code>, <code>command</code>, <code>data</code>, <code>datalist</code>, <code>details</code>, <code>embed</code>, <code>figcaption</code>, <code>figure</code>, <code>footer</code>, <code>header</code>, <code>keygen</code>, <code>mark</code>, <code>meter</code>, <code>nav</code>, <code>output</code>, <code>progress</code>, <code>rp</code>, <code>rt</code>, <code><a href="/wiki/Ruby_(annotation_markup)" title="Ruby (annotation markup)" class="mw-redirect">ruby</a></code>, <code>section</code>, <code>source</code>, <code>summary</code>, <code>time</code>, <code>track</code>, <code>video</code>, <code>wbr</code></li>
<li>New types of form controls: <code>dates and times</code>, <code>email</code>, <code>url</code>, <code>search</code>, <code>number</code>, <code>range</code>, <code>tel</code>, <code>color</code><sup id="cite_ref-76" class="reference"><a href="#cite_note-76">[75]</a></sup></li>
<li>New <a href="/wiki/HTML#Attributes" title="HTML">attributes</a>: <code>charset</code> (on <code>meta</code>), <code>async</code> (on <code>script</code>)</li>
<li>Global attributes (that can be applied for every element): <code>id</code>, <code>tabindex</code>, <code>hidden</code>, <code>data-*</code> (custom data attributes)</li>
<li>Deprecated elements will be dropped altogether: <code>acronym</code>, <code>applet</code>, <code>basefont</code>, <code>big</code>, <code>center</code>, <code>dir</code>, <code>font</code>, <code>frame</code>, <a href="/wiki/Framing_(World_Wide_Web)" title="Framing (World Wide Web)"><code>frameset</code></a>, <code>isindex</code>, <code>noframes</code>, <code>strike</code>, <code>tt</code></li>
</ul>
<p>dev.w3.org provides the latest <i>Editors Draft</i> of "HTML5 differences from HTML 4",<sup id="cite_ref-77" class="reference"><a href="#cite_note-77">[76]</a></sup> which provides a complete outline of additions, removals and changes between HTML5 and HTML 4.</p>
<h2><span id="Logo" class="mw-headline">Logo</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=13" title="Edit section: Logo">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div style="width:122px;" class="thumbinner"><a href="/wiki/File:HTML5_logo_and_wordmark.svg" class="image"><img alt src="//upload.wikimedia.org/wikipedia/commons/thumb/6/61/HTML5_logo_and_wordmark.svg/120px-HTML5_logo_and_wordmark.svg.png" width="120" height="120" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/6/61/HTML5_logo_and_wordmark.svg/180px-HTML5_logo_and_wordmark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/6/61/HTML5_logo_and_wordmark.svg/240px-HTML5_logo_and_wordmark.svg.png 2x" data-file-width="512" data-file-height="512" class="thumbimage"></a>
<div class="thumbcaption">
<div class="magnify"><a href="/wiki/File:HTML5_logo_and_wordmark.svg" title="Enlarge" class="internal"></a></div>
The W3C HTML5 logo</div>
</div>
</div>
<p>On 18 January 2011, the W3C introduced a logo to represent the use of or interest in HTML5. Unlike other badges previously issued by the W3C, it does not imply validity or conformance to a certain standard. As of 1 April 2011, this logo is official.<sup id="cite_ref-logo-faq_78-0" class="reference"><a href="#cite_note-logo-faq-78">[77]</a></sup></p>
<p>When initially presenting it to the public, the W3C announced the HTML5 logo as a "general-purpose visual identity for a broad set of open web technologies, including HTML5, <a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a>, SVG, <a href="/wiki/Web_Open_Font_Format" title="Web Open Font Format">WOFF</a>, and others".<sup id="cite_ref-waspopenletter_79-0" class="reference"><a href="#cite_note-waspopenletter-79">[78]</a></sup> Some web standard advocates, including The Web Standards Project, criticized that definition of "HTML5" as an umbrella term, pointing out the blurring of terminology and the potential for miscommunication.<sup id="cite_ref-waspopenletter_79-1" class="reference"><a href="#cite_note-waspopenletter-79">[78]</a></sup> Three days later, the W3C responded to community feedback and changed the logo's definition, dropping the enumeration of related technologies.<sup id="cite_ref-80" class="reference"><a href="#cite_note-80">[79]</a></sup> The W3C then said the logo "represents HTML5, the cornerstone for modern Web applications".<sup id="cite_ref-logo-faq_78-1" class="reference"><a href="#cite_note-logo-faq-78">[77]</a></sup></p>
<h2><span id="Digital_rights_management" class="mw-headline">Digital rights management</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5&amp;action=edit&amp;section=14" title="Edit section: Digital rights management">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Industry players including the <a href="/wiki/BBC" title="BBC">BBC</a>, <a href="/wiki/Google" title="Google">Google</a>, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>, and <a href="/wiki/Netflix" title="Netflix">Netflix</a> have been lobbying for the inclusion of <a href="/wiki/Encrypted_Media_Extensions" title="Encrypted Media Extensions">Encrypted Media Extensions</a> (EME),<sup id="cite_ref-81" class="reference"><a href="#cite_note-81">[80]</a></sup><sup id="cite_ref-82" class="reference"><a href="#cite_note-82">[81]</a></sup><sup id="cite_ref-83" class="reference"><a href="#cite_note-83">[82]</a></sup><sup id="cite_ref-84" class="reference"><a href="#cite_note-84">[83]</a></sup><sup id="cite_ref-85" class="reference"><a href="#cite_note-85">[84]</a></sup> a form of <a href="/wiki/Digital_rights_management" title="Digital rights management">digital rights management</a> (DRM), into the HTML5 standard. As of the end of 2012 and the beginning of 2013, 27 organisations<sup id="cite_ref-86" class="reference"><a href="#cite_note-86">[85]</a></sup> including the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a><sup id="cite_ref-87" class="reference"><a href="#cite_note-87">[86]</a></sup> have started a campaign against including digital rights management in the HTML5 standard.<sup id="cite_ref-88" class="reference"><a href="#cite_note-88">[87]</a></sup><sup id="cite_ref-89" class="reference"><a href="#cite_note-89">[88]</a></sup> However, in late September 2013, the W3C <a href="/wiki/HTML_Working_Group" title="HTML Working Group">HTML Working Group</a> decided that Encrypted Media Extensions, a form of DRM, was "in scope" and will potentially be included in the HTML 5.1 standard.<sup id="cite_ref-90" class="reference"><a href="#cite_note-90">[89]</a></sup><sup id="cite_ref-lowering-your-standards_91-0" class="reference"><a href="#cite_note-lowering-your-standards-91">[90]</a></sup> <a href="/wiki/WHATWG" title="WHATWG">WHATWG</a>'s "HTML Living Standard" continued to be developed without DRM-enabled proposals.<sup id="cite_ref-lowering-your-standards_91-1" class="reference"><a href="#cite_note-lowering-your-standards-91">[90]</a></sup></p>
<p>Manu Sporny, a member of the <a href="/wiki/W3C" title="W3C" class="mw-redirect">W3C</a>, said that EME will not solve the problem it's supposed to address.<sup id="cite_ref-92" class="reference"><a href="#cite_note-92">[91]</a></sup> Opponents point out that EME itself is just an architecture for a DRM plug-in mechanism.<sup id="cite_ref-93" class="reference"><a href="#cite_note-93">[92]</a></sup></p>
<p>The initial enablers for DRM in HTML5 were Google<sup id="cite_ref-94" class="reference"><a href="#cite_note-94">[93]</a></sup> and Microsoft.<sup id="cite_ref-95" class="reference"><a href="#cite_note-95">[94]</a></sup> Supporters also include Adobe.<sup id="cite_ref-96" class="reference"><a href="#cite_note-96">[95]</a></sup> On 14 May 2014, <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla</a> announced plans to support EME in <a href="/wiki/Mozilla_Firefox" title="Mozilla Firefox" class="mw-redirect">Firefox</a>, the last major browser to avoid DRM.<sup id="cite_ref-gal_97-0" class="reference"><a href="#cite_note-gal-97">[96]</a></sup><sup id="cite_ref-baker_98-0" class="reference"><a href="#cite_note-baker-98">[97]</a></sup> Calling it "a difficult and uncomfortable step", Andreas Gal of Mozilla explained that future versions of Firefox would remain open source but ship with a sandbox designed to run a content decryption module developed by Adobe.<sup id="cite_ref-gal_97-1" class="reference"><a href="#cite_note-gal-97">[96]</a></sup> While promising to "work on alternative solutions", Mozilla's Executive Chair <a href="/wiki/Mitchell_Baker" title="Mitchell Baker">Mitchell Baker</a> stated that a refusal to implement EME would have accomplished little more than convincing many users to switch browsers.<sup id="cite_ref-baker_98-1" class="reference"><a href="#cite_note-baker-98">[97]</a></sup> This decision was condemned by <a href="/wiki/Cory_Doctorow" title="Cory Doctorow">Cory Doctorow</a> and the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a>.<sup id="cite_ref-99" class="reference"><a href="#cite_note-99">[98]</a></sup><sup id="cite_ref-100" class="reference"><a href="#cite_note-100">[99]</a></sup></p>
